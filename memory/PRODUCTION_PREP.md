# BUMP — Production Prep Guide

Use this guide once you're ready to ship from MVP → production.

---

## 1. Resend (email) — domain verification

**Current state:** Resend is in **sandbox mode** — only delivers to `wissambizri961@gmail.com`.

### Steps to verify a domain

1. Buy a domain (Namecheap / Cloudflare Registrar / Porkbun — ~$10/yr).
2. https://resend.com/domains → **Add Domain** → enter e.g. `bump.app`.
3. Resend will show DNS records to add (4–6 total: SPF, DKIM, MX, DMARC):
   - **SPF**: `TXT @ "v=spf1 include:_spf.resend.com ~all"`
   - **DKIM**: 3 CNAME records starting with `resend._domainkey...`
   - **MX**: optional (for inbound)
   - **DMARC**: `TXT _dmarc "v=DMARC1; p=quarantine; rua=mailto:dmarc@bump.app"`
4. Add the records in your DNS provider's dashboard.
5. Back in Resend → click **Verify** (DNS propagation: 5min → 24h).
6. Once verified, update `/app/backend/.env`:
   ```
   RESEND_FROM_EMAIL="auth@bump.app"
   DEMO_MODE=0    # stops leaking OTP codes in API responses
   ```
7. Restart backend: `sudo supervisorctl restart backend`.

## 4. EAS Build setup

1. Install: `npm install -g eas-cli`
2. `cd /app/frontend && eas login`
3. `eas build:configure` (creates `eas.json`)
4. **Internal preview build (for testing):**
   ```
   eas build --platform ios --profile preview
   eas build --platform android --profile preview
   ```
5. **Production:**
   ```
   eas build --platform all --profile production
   ```
6. **Submit to stores:**
   ```
   eas submit --platform ios
   eas submit --platform android
   ```

> Requires: paid Apple Developer ($99/yr) and Google Play Developer ($25 one-time).

## 5. Environment variables — set in EAS

Don't commit secrets! In EAS dashboard → Project → Secrets, set:
- `GOOGLE_PLACES_API_KEY`
- `TWILIO_ACCOUNT_SID`
- `TWILIO_AUTH_TOKEN`
- `RESEND_API_KEY`
- `JWT_SECRET` — generate a new strong one for production (`openssl rand -hex 32`)
- `DEMO_MODE=0`

For the backend (hosted separately on Railway/Render/Fly.io):
- Use the same env vars in your hosting platform's dashboard.

---

## 6. MongoDB production

Migrate from local Mongo to **MongoDB Atlas** (free tier):
1. https://cloud.mongodb.com → create cluster → M0 free tier
2. Add IP allowlist (0.0.0.0/0 for now, restrict later)
3. Create user → copy connection string
4. Update `MONGO_URL` in backend production env

---

## 7. Backend hosting

Recommended: **Railway** (`railway.app`) for FastAPI + Mongo:
1. Connect GitHub → import the repo
2. Set start command: `cd backend && uvicorn server:app --host 0.0.0.0 --port $PORT`
3. Set env vars (see Step 5)
4. Deploy → get public URL → update `EXPO_PUBLIC_BACKEND_URL` in frontend `.env`
